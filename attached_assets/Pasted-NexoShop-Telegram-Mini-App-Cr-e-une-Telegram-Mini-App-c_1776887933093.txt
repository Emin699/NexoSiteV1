NexoShop Telegram Mini App
Crée une Telegram Mini App complète pour ma boutique NexoShop. Voici le cahier des charges détaillé.

🎯 Stack technique demandé
Frontend : React 19 + Vite + TypeScript + TailwindCSS (mobile-first, dark mode)
Backend : Node.js + Express + TypeScript
Base de données : PostgreSQL + Drizzle ORM
Bot Telegram : node-telegram-bot-api en mode polling
Le SDK Telegram WebApp doit être chargé : https://telegram.org/js/telegram-web-app.js
🔐 Contraintes de sécurité critiques
L'app DOIT être accessible UNIQUEMENT depuis Telegram.
Si quelqu'un ouvre l'URL dans Chrome/Safari sans initData Telegram → afficher un écran "❌ Cette application est accessible uniquement depuis le bot Telegram NexoShop" + bouton "Ouvrir le bot" qui redirige vers https://t.me/MonBotUsername.
Côté backend : valider obligatoirement le initData Telegram via HMAC-SHA256 sur toutes les routes API (sauf /health). Pas de fallback dev en production.
Algo : secret_key = HMAC_SHA256("WebAppData", BOT_TOKEN) puis expected_hash = HMAC_SHA256(secret_key, sorted_data_check_string).
L'ID Telegram de l'utilisateur doit être extrait du initDataUnsafe.user.id validé côté serveur — jamais du frontend.
Toute opération sensible (achat, recharge, conversion points) passe par le backend authentifié.
🤖 Comportement du bot Telegram
Commande /start
Le bot doit répondre avec :

Une image de bienvenue (banner/logo NexoShop — utilise un placeholder hébergé pour l'instant, je remplacerai)
Un message court : "👋 Bienvenue sur NexoShop !\nClique sur le bouton ci-dessous pour accéder à la boutique."
Un seul bouton inline de type web_app :
Texte : 🛍️ Accéder à la boutique
URL : process.env.MINI_APP_URL
Toutes les autres anciennes commandes utilisateur (/menu, /solde, /panier...) doivent être supprimées. Garde uniquement les commandes admin (/admin, /addbalance, etc.).

Système de référence (parrainage)
/start ref_<userId> → enregistre que <userId> a parrainé le nouvel arrivant.

Section 2 : Recharger
3 cartes de méthode de paiement :

🪙 Litecoin (LTC) — automatique
Min : 5€
Frais réseau : +2% sur le taux de change
Flow : choix montant (5/10/20/30/50/autre) → conversion EUR→LTC via API CoinGecko → affichage adresse + montant LTC → input "TX Hash" → vérification via Blockchair (1 confirmation requise) → crédit auto
🅿️ PayPal — semi-automatique
Min : 5€
Anti-spam : max 3 tentatives/heure, auto-ban si >5 paiements PENDING
Flow : choix montant → 2 modes selon PAYPAL_API_ENABLED :
API active : génère référence aléatoire (ex: "pizza weekend a1b2c3d4"), user envoie via PayPal.me avec cette note, polling auto vérifie
API inactive : user paie puis envoie un screenshot → notifie admin → boutons Valider/Attendre/Refuser
💳 SumUp (CB / Apple Pay) — automatique
Génère un lien de paiement SumUp
Polling vérifie l'historique pour transaction SUCCESSFUL correspondant
